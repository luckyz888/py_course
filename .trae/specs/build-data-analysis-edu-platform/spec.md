# 商务数据分析在线教育平台 Spec

## Why
商务数据分析与应用专业的学生缺乏一个集课程学习、代码实践、测评考核于一体的在线平台。现有资源分散，无法提供从学、练到测的完整闭环体验，也缺少激励机制来维持学习动力。

## What Changes
- 构建一个纯前端的 SPA 在线教育平台，使用 React + Vite + TypeScript 开发
- 基于 Pyodide（WebAssembly）在浏览器内运行 Python 代码，实现交互式编程练习
- 设计面向商务数据分析与应用专业的完整课程体系（6大模块，覆盖 Python 基础到商务实战）
- 实现学-练-测三位一体的学习流程
- 实现成就激励系统（徽章、积分、连续学习、等级）
- 所有数据使用 localStorage/IndexedDB 持久化，无需后端服务器
- 部署到 Cloudflare Pages（免费用户方案）

## Impact
- Affected specs: 全新项目，无已有规格
- Affected code: 全新代码库

## ADDED Requirements

### Requirement: 课程体系
系统 SHALL 提供面向商务数据分析与应用专业的完整课程体系，包含以下6大模块：

1. **Python编程基础** — 变量与数据类型、运算符、流程控制、函数、模块与包、文件操作
2. **数据处理与分析** — NumPy基础、Pandas数据结构、数据清洗、数据合并与重塑、数据聚合与分组
3. **数据可视化** — Matplotlib基础绘图、Seaborn高级可视化、ECharts交互图表、商业报表图表设计
4. **统计分析** — 描述性统计、概率分布、假设检验、相关与回归分析、时间序列分析
5. **商务智能与报表** — Excel数据处理、数据透视分析、商业指标体系、自动化报表生成
6. **数据挖掘与商务实战** — 分类与预测、聚类分析、关联规则、推荐系统基础、商务案例分析

每个模块包含：课程说明、知识点列表、理论讲解、代码示例、练习题、测评题。

#### Scenario: 学生浏览课程
- **WHEN** 学生进入课程页面
- **THEN** 显示6大模块，每个模块可展开查看章节和知识点

#### Scenario: 学生进入知识点学习
- **WHEN** 学生点击某个知识点
- **THEN** 显示理论讲解、代码示例和互动练习

### Requirement: 互动式学习模块
系统 SHALL 提供基于 Pyodide 的浏览器内 Python 运行环境，支持：

- 代码编辑器（语法高亮、自动缩进）
- 一键运行 Python 代码并显示输出
- 预置代码模板和示例
- 运行结果可视化（支持图表渲染）
- 代码错误提示

#### Scenario: 学生运行代码
- **WHEN** 学生在代码编辑器中编写代码并点击运行
- **THEN** 在输出区域显示运行结果或错误信息

#### Scenario: 学生运行可视化代码
- **WHEN** 学生运行包含 matplotlib/seaborn 的代码
- **THEN** 在输出区域渲染图表

### Requirement: 学-练-测学习流程
系统 SHALL 提供三位一体的学习流程：

**学**：理论知识讲解 + 代码示例演示
**练**：交互式编程练习，提供题目描述、初始代码、提示、参考答案
**测**：章节测评，包含选择题、填空题、编程题，自动评分

#### Scenario: 学生完成练习
- **WHEN** 学生提交编程练习的代码
- **THEN** 系统运行代码并与预期输出比对，给出通过/未通过反馈

#### Scenario: 学生完成测评
- **WHEN** 学生提交章节测评
- **THEN** 系统自动评分并显示成绩、正确答案和解析

### Requirement: 成就激励系统
系统 SHALL 提供以下激励机制：

- **积分系统**：完成学习+10分，完成练习+20分，通过测评+50分
- **徽章系统**：首次完成课程、连续学习7天、满分通过测评、完成全部课程等
- **连续学习**：记录每日学习，显示连续天数和最长连续记录
- **等级系统**：根据积分划分等级（入门→初级→中级→高级→专家）
- **学习进度**：可视化展示各模块完成百分比

#### Scenario: 学生获得徽章
- **WHEN** 学生达成某个成就条件
- **THEN** 弹出徽章获得通知，并在成就页面展示

#### Scenario: 学生查看学习进度
- **WHEN** 学生进入个人中心
- **THEN** 显示总积分、等级、连续学习天数、各模块完成进度

### Requirement: Cloudflare Pages 部署
系统 SHALL 适配 Cloudflare Pages 免费用户方案：

- 纯静态 SPA，无需后端服务器
- 所有数据使用 localStorage/IndexedDB 客户端存储
- Pyodide 从 CDN 加载，不打包进构建产物
- 构建产物体积优化（代码分割、懒加载）
- 支持客户端路由（SPA 模式）

#### Scenario: 部署到 Cloudflare Pages
- **WHEN** 执行 `npm run build` 后部署到 Cloudflare Pages
- **THEN** 网站可正常访问，所有功能可用

### Requirement: 响应式设计
系统 SHALL 支持桌面端和平板端访问，最小适配宽度 768px。

#### Scenario: 平板端访问
- **WHEN** 用户使用平板设备访问
- **THEN** 页面布局自适应，内容可正常浏览和操作

### Requirement: 数据持久化
系统 SHALL 使用浏览器本地存储保存所有用户数据：

- 学习进度
- 练习和测评记录
- 积分和成就
- 用户偏好设置

#### Scenario: 用户关闭浏览器后重新访问
- **WHEN** 用户关闭浏览器后再次打开平台
- **THEN** 所有学习进度和成就数据保留完好
